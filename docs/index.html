Architecture

Single process. Event bus. Pluggable channels.

One Node.js process runs everything. Channels and the agent loop are decoupled through an async message bus.

Message Bus

Channels and the agent loop are fully decoupled through async inbound/outbound queues. Channels publish; the agent consumes, runs a turn, and publishes replies back.

Pluggable Channels

Each channel implements the same adapter interface: start, stop, send, editMessage. The channel manager owns lifecycle and routes outbound messages to the right adapter.

Command Interception

Slash commands (/session, /model, /config) are intercepted before reaching the LLM and execute instantly without spending tokens.

Streaming Updates

During a turn, tool call progress and streaming text are pushed back as editable messages that get replaced with the final response.

Key Files

index.ts — Boots the runtime: config, bus, agent, channels, heartbeat
core/agent-loop.ts — Consumes inbound messages, runs LLM turns, publishes replies
core/bus.ts — Async message bus with inbound/outbound queues
channels/manager.ts — Owns channel lifecycle and outbound dispatch
core/client-factory.ts — Creates Claude or Codex model client based on config
core/session-store.ts — Persists conversation sessions to a JSON file

FAQ

Common questions

Everything you need to know before getting started.

What is claude-pipe?
claude-pipe is a lightweight bridge that connects your chat apps (Discord, CLI) to a real Claude Code session running on your machine. You send a message from your phone and get the full Claude Code experience — file editing, shell commands, web search, Google integrations — without opening a terminal.
Is it the same as Claude Code?
Yes. claude-pipe doesn't limit or wrap Claude — it runs the actual Claude Code CLI (or Codex CLI) under the hood with the same tools, skills, and context. It's just a different front door: your phone instead of a terminal.
What do I need to run it?
Node.js 20+, a Claude Code subscription (or Codex CLI), and a Discord bot token. The interactive setup wizard handles configuration. Your machine needs to stay running since the agent runs locally.
Does my data leave my machine?
Messages flow through the Discord API (as with any bot), but the agent itself runs entirely on your machine. Files, code, and credentials never leave your server. Sessions are stored in a local JSON file.
Can I restrict who can use the bot?
Yes. Each channel has an allow-list setting. Only user IDs in the list can interact with the bot. If the list is empty, everyone is allowed. The setup wizard helps you configure this.
Which LLM providers are supported?
claude-pipe supports Claude Code (via the Claude CLI) and OpenAI's Codex CLI. You can switch between them using the /model slash command or by changing the provider in settings. The model client is pluggable, so adding new providers is straightforward.
